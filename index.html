Multimodal detection of ovarian cancer

"Early detection of ovarian cancer can be significantly improved by integrating clinical biomarker data (such as CA-125 levels, age, family history, and hormonal factors) with medical imaging features (ultrasound/CT/MRI) using AI-ML models. A multimodal machine learning approach that learns patterns jointly from structured clinical parameters and image-based tumor characteristics will achieve higher prediction accuracy, sensitivity, and earlier risk identification compared to models relying on a single data source."

Problem Statement

Ovarian cancer is the fifth leading cause of cancer deaths among women globally. Approximately 70% of all ovarian cancer cases are diagnosed only in Stage III or Stage IV, where the five-year survival rate drops below 30%. In contrast, when detected at Stage I, the survival rate exceeds 90%, underscoring the life-saving potential of early diagnosis.

Current clinical diagnostic pathways rely heavily on the CA-125 blood biomarker test and transvaginal ultrasound imaging. These methods, while useful, are analyzed separately and suffer from inherent limitations: CA-125 is non-specific and can be elevated in benign conditions, while ultrasound interpretation is highly dependent on radiologist expertise. No existing clinical standard integrates both modalities in a structured, automated fashion.

This project proposes and implements an AI-Driven Multimodal Learning System that fuses structured clinical biomarker data with unstructured histopathological imaging data to enable earlier and more accurate ovarian cancer detection. The dual-branch architecture achieves state-of-the-art performance through specialized ensemble methods and deep transfer learning.

Literature Review / Market Research

  1. Liu et al., 2019: Highlighted the importance of key biomarkers like CA-125, HE4, and other clinical factors in improving ML-based prediction models.
  2. Zhang et al., 2020: Compared machine learning models such as Random Forest and XGBoost using biomarker data for ovarian cancer prediction and found improved accuracy over traditional statistical methods.
  3. Nunes et al., 2021: Integrated clinical and imaging data using deep learning techniques to support diagnosis, showing that combining multiple data types improves performance.
  4. Boehm et al., 2022: Developed a multimodal ML approach for risk stratification and used feature importance and image heatmaps to improve interpretability of predictions.

Research Gap / Innovation

Research Gap

  • Most existing approaches rely on highly complex datasets and expensive imaging techniques
  • Research-level deep learning systems are difficult to implement in real clinical settings
  • Many models are trained on limited or specialized populations
  • Current solutions are not designed for simple, scalable deployment

Innovation

  • Practical and accessible multimodal AI-ML model using commonly available biomarkers
  • Emphasizes simplicity, interpretability, and real-world usability
  • Compares multiple ML techniques for comprehensive evaluation
  • Highlights important features for transparent clinical decision-making

System Methodology

Dataset / Input

This project uses publicly available datasets from Kaggle:

  • Biomarker Dataset: Kaggle Ovarian Cancer Dataset (238 patients, 48 clinical features)
  • Imaging Dataset: The Cancer Genome Atlas (TCGA) Ovarian Histological Images
    Total: 33,110 images (19,747 Tumor, 13,363 Non-Tumor)

Data Files Used:

Biomarker Features: RBC, HGB, PLT, CA125, HE4, CA19-9, CEA, AFP + 40 others
Train-Test Split: 70% training (160 patients), 30% testing (71 patients)
Image Distribution: 5,000 image subset (4,000 training, 1,000 validation)
Image Preprocessing: 224×224 resize, pixel normalization [0,1], augmentation applied
Class Distribution: ~62% Non-cancerous, ~38% Cancerous (stratified split)

Model / Architecture

A multimodal AI-ML workflow is used to combine clinical data and image-based inputs.

Dual-Branch Architecture

Branch 1 - Biomarker: Random Forest (200 trees) + XGBoost
Branch 2 - Imaging: DenseNet121 (fine-tuned on TCGA images)
Fusion Strategy: Late-fusion via probability concatenation
